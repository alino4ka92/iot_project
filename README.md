# Система мониторинга показателей здоровья крупного рогатого скота

Проект предназначен для повышения рентабельности молочных ферм посредством раннего выявления заболеваний у животных. Система обеспечивает сбор, передачу, обработку и визуализацию телеметрических данных.

## Архитектура системы

- **Источники данных:** Эмуляция носимых IoT-устройств через мобильное приложение "IoT MQTT Panel".
- **Брокер сообщений:** Eclipse Mosquitto (MQTT).
- **Бэкенд:** REST API на базе Python (FastAPI, Paho MQTT) для агрегации данных, расчета вероятности заболеваний и управления оповещениями.
- **База данных:** Time-Series СУБД InfluxDB для хранения метрик.
- **Визуализация:** Grafana с доступом "только для чтения".
- **Клиентское приложение:** Веб-интерфейс (HTML/JS/CSS), интегрирующий метрики Grafana и картографические данные.

## Структура репозитория

- `infrastructure/` — конфигурация контейнеризации (Docker Compose) для базовых сервисов.
- `backend_service/` — исходный код серверной части (обработка MQTT, маршрутизация, бизнес-логика).
- `mobile_dashboard/` — конфигурационный файл для настройки мобильного эмулятора.
- `web_app/` — статические файлы клиентского веб-приложения.

## Руководство по развертыванию

### 1. Запуск серверной инфраструктуры

Требуется установленный Docker и Docker Compose.

```bash
cd infrastructure
docker-compose up -d --build
```

### 2. Доступ к веб-интерфейсу

1. Клиентское приложение доступно по адресу: [http://localhost:3001](http://localhost:3001).
2. Дашборды Grafana интегрированы в интерфейс автоматически (используется механизм `provisioning`). Прямой доступ к Grafana: [http://localhost:3000](http://localhost:3000).

### 3. Эмуляция передачи данных

1. Установите приложение **IoT MQTT Panel** на мобильное устройство.
2. Импортируйте конфигурацию из файла `mobile_dashboard/iot_mqtt_panel_cows.json`.
3. В настройках подключения укажите IP-адрес хоста (ПК) в локальной сети вместо `YOUR_IP_ADDRESS_HERE`.
4. Изменение параметров температуры и активности в приложении инициирует отправку данных на брокер. Сервер обработает метрики и обновит дашборды в режиме реального времени.

## Политика безопасности

Клиентское приложение не имеет прямого доступа к СУБД InfluxDB. Операции записи выполняются исключительно через изолированный серверный компонент. Интеграция Grafana настроена с правами только на чтение, что предотвращает несанкционированную модификацию данных.
