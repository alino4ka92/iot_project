# Учебный IoT Проект: Мониторинг жизненных показателей коров

Проект направлен на решение проблемы низкой маржинальности молочных ферм путём раннего выявления заболеваний коров. В проекте эмулируется сбор данных с датчиков, их передача, обработка и визуализация.

## Архитектура

- **Источники данных:** Мобильное приложение "IoT MQTT Panel" на смартфоне (используется для эмуляции носимого датчика коровы).
- **Брокер сообщений:** Eclipse Mosquitto (MQTT).
- **Бэкенд:** Кастомный Python скрипт (FastAPI + Paho MQTT в `backend_service/`), который слушает MQTT, рассчитывает вероятность заболевания и пишет данные в базу.
- **База данных:** InfluxDB (Time-Series база данных для хранения метрик).
- **Визуализация (Backend):** Grafana (настроена для безопасного read-only доступа к InfluxDB).
- **Фронтенд (Web App):** Кастомный HTML/JS/CSS интерфейс (`web_app/`), в который встраиваются графики из Grafana.

## Структура папок

- `infrastructure/` — Конфигурации для запуска через Docker (Mosquitto, InfluxDB, Grafana).
- `backend_service/` — Код Python сервера (парсинг MQTT, логика болезни, запись в БД).
- `mobile_dashboard/` — JSON-конфигурация для смартфона (IoT MQTT Panel).
- `web_app/` — Исходный код пользовательского веб-интерфейса.

## Как запустить проект локально

### Шаг 1. Запуск инфраструктуры и бэкенда

Убедитесь, что у вас установлен Docker и Docker Compose.

1. Откройте терминал и перейдите в папку `infrastructure`:
   ```bash
   cd infrastructure
   ```
2. Запустите сервисы в фоновом режиме:
   ```bash
   docker-compose up -d --build
   ```

*(Docker скачает нужные образы InfluxDB, Mosquitto, Grafana, соберет наш Python бэкенд и запустит всё вместе).*

### Шаг 2. Запуск Web App и Grafana

1. Откройте файл `web_app/index.html` в вашем браузере (просто двойным кликом). Вы увидите кастомный интерфейс нашего приложения.
2. Графики (Температура, Активность, Вероятность заболевания) подгрузятся из Grafana автоматически (они встроены через `<iframe>`), так как мы преднастроили дашборд через механизм `provisioning`.
3. При желании вы можете посмотреть оригинальный дашборд в самой Grafana по адресу [http://localhost:3000](http://localhost:3000). Логин по умолчанию не требуется (анонимный доступ).

### Шаг 4. Отправка данных со смартфона

1. Установите на телефон приложение **IoT MQTT Panel** (или аналогичное).
2. Зайдите в приложение и импортируйте конфигурацию из файла `mobile_dashboard/iot_mqtt_panel_cows.json`.
3. В настройках подключения (Connection) в приложении **обязательно измените IP-адрес** (host) с `YOUR_IP_ADDRESS_HERE` на локальный IP-адрес вашего компьютера в Wi-Fi сети (например, `192.168.1.5`).
4. Подключитесь к брокеру и подвигайте ползунки Температуры и Активности.
5. Наш `backend_service` автоматически считает эти данные, вычислит вероятность заболевания и запишет в InfluxDB, после чего графики в веб-приложении мгновенно обновятся.

## Безопасность
Фронтенд и внешние пользователи **не имеют** прямого доступа к базе данных InfluxDB. 
Запись данных осуществляется исключительно через защищенный бэкенд-сервис, а Grafana подключается к базе с правами только для чтения, что исключает возможность случайного удаления данных.
