# Учебный IoT-проект: Система мониторинга здоровья коров

Данный проект разработан для снижения финансовых потерь на молочных фермах за счет раннего выявления заболеваний у животных. Система обеспечивает сбор данных с датчиков, их анализ и отображение на веб-дашборде для оперативного контроля за состоянием стада.

## Архитектура системы

- **Источники данных:** Мобильное приложение "IoT MQTT Panel" (используется для эмуляции IoT-датчика на животном).
- **Брокер сообщений:** Eclipse Mosquitto (MQTT).
- **Бэкенд:** Сервер на базе Python (FastAPI + Paho MQTT). Отвечает за сбор телеметрии, расчет вероятности заболеваний и формирование уведомлений.
- **База данных:** InfluxDB (оптимальное решение для хранения метрик временных рядов).
- **Визуализация:** Grafana (используется для построения графиков, настроена в режиме "только чтение").
- **Веб-интерфейс:** Реализован на HTML/JS/CSS, включает в себя встроенные дашборды Grafana и отображение координат на базе Яндекс.Карт.

## Структура папок

- `infrastructure/` — файлы для настройки и запуска сервисов через Docker Compose.
- `backend_service/` — исходный код бэкенда (обработка MQTT, бизнес-логика, API).
- `mobile_dashboard/` — конфигурация для мобильного приложения-эмулятора.
- `web_app/` — исходные файлы пользовательского веб-интерфейса.

## Руководство по запуску

### 1. Запуск инфраструктуры и сервера

Для работы проекта необходимы Docker и Docker Compose. Откройте терминал и выполните команды:

```bash
cd infrastructure
docker-compose up -d --build
```

### 2. Доступ к интерфейсу

1. Веб-приложение доступно по локальному адресу: [http://localhost:3001](http://localhost:3001).
2. Графики подгружаются автоматически. Для прямого доступа к интерфейсу Grafana можно использовать адрес: [http://localhost:3000](http://localhost:3000).

### 3. Эмуляция отправки данных

1. Установите на смартфон приложение **IoT MQTT Panel**.
2. Импортируйте конфигурацию из файла `mobile_dashboard/iot_mqtt_panel_cows.json`.
3. В настройках подключения укажите локальный IP-адрес вашего компьютера в локальной сети (вместо `YOUR_IP_ADDRESS_HERE`).
4. При изменении параметров температуры и активности в приложении данные будут отправляться на сервер, а графики в веб-интерфейсе обновятся в реальном времени.

## Меры безопасности

Фронтенд не имеет прямого доступа к базе данных InfluxDB. Запись данных осуществляется исключительно через бэкенд-сервис, а Grafana подключается с правами "только для чтения". Это предотвращает случайное удаление или изменение накопленной истории.

## Реализованные функции

- **Мониторинг в реальном времени**: получение данных о температуре, активности и координатах по легковесному протоколу MQTT.
- **Аналитика и уведомления**: алгоритмический расчет вероятности заболевания коров на основе полученных данных и автоматическая генерация алертов.
- **Визуализация данных**: отображение истории показателей на графиках Grafana и вывод местоположения стада на интерактивной карте.
- **Динамические настройки**: возможность изменять пороговые значения (для температуры, активности и частоты уведомлений) через веб-интерфейс без перезапуска сервера.
- **Журнал событий**: сохранение и вывод истории критических уведомлений для анализа состояния стада.

## Обоснование выбора технологий

В проекте использованы инструменты, которые хорошо зарекомендовали себя в сфере интернета вещей (IoT):

- **Python и FastAPI**: Python удобен для работы с данными и написания бизнес-логики, а фреймворк FastAPI позволяет быстро разработать производительный REST API.
- **Eclipse Mosquitto (MQTT)**: протокол обмена сообщениями, который является стандартом для IoT-систем. Он оптимален для частой передачи небольших пакетов данных от сенсоров.
- **InfluxDB**: специализированная Time-Series база данных. Она отлично подходит для записи временных рядов (показатели датчиков с привязкой ко времени) и быстрого построения графиков.
- **Grafana**: платформа для визуализации и мониторинга. Идеально интегрируется с InfluxDB и позволяет без лишнего кода создавать красивые дашборды, которые легко встраиваются в веб-приложение.
- **Docker и Docker Compose**: используются для контейнеризации приложения, что решает проблемы с системными зависимостями и позволяет развернуть весь проект одной командой.
